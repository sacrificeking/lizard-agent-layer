# Design System

## Rules

- Prefer the target project's design contract over invented styling.
- Use semantic tokens where available.
- Keep page layout, typography, and spacing consistent with existing components.
- Do not edit the design contract unless the user asks for a design-system change.
- Verify responsive behavior for user-facing UI changes.
