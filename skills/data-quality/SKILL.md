# Data Quality

## Rules

- Distinguish unknown from zero or safe.
- Keep source URLs and timestamps when presenting facts.
- Add assertions for critical seed or migration output.
- Do not let fallback data imply confidence it does not have.
- Prefer field-level provenance for high-stakes data.
